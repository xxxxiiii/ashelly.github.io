vue-vuex状态管理模式

发表于 2020-03-27 更新于 2020-04-05 分类于 学习笔记

Vuex是一个专为Vue.js应用程序开发的状态管理模式，它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化

那么，状态管理到底是什么？

状态管理模式、集中式存储管理看起来有点难理解，其实，我们可以简单地将其看成把需要多个组件共享的变量全部存储在一个对象里面，然后，将这个对象放在顶层的Vue实例中，让其他组件可以使用。

我们自己可以封装一个对象来管理吗？

当然可以，只是Vuejs带给我们最大的便利就是响应式，而我们自己封装实现一个对象保证它里面所有的属性做到响应式可能会稍微麻烦一点，所以用Vuex就可以了，不要重复造轮子。

多界面状态管理

  • Vue已经帮我们做好了单界面状态管理，多界面状态管理就需要用到vuex

  • vuex是全局单例模式，可以理解为大管家。我们要做的就是将共享的状态抽取出来，交给大管家，统一进行管理，之后，每个视图按照规定进行访问和修改等操作。

  • 在Vue项目开发中，需要监控项目中得各种值，为了提高效率，Vue提供了一款浏览器扩展——VueDevtools。所以我们要在浏览器安装VueDevtools插件

  • Vuex状态管理图例

    • 修改state时一定要通过mutation修改，这样devtools才能跟踪变化；
    • 如果有异步操作不要通过mutation操作，devtools只能跟踪同步操作，最好通过action操作
vuex核心概念：state，getters，mutation，action，module
  • state(单一状态树)

    • 管理应用层的全部状态，可以直接找到某个状态的片段，在之后的维护和调试过程中可以非常方便地管理和维护。

  • getters

    • 类似于单界面中的计算属性。

    • 需要传参时可以返回带参回调函数。

  • mutation(状态更新)

    • 主要包含两部分：事件类型、回调函数，该回调函数的第一个参数是state

    • 参数被称为是mutation的负荷(payload)

    • 提交风格：1. 通过commit 提交字符串 => this.$store.commit(“”)；2. 通过commit 提交 对象

    • mutation响应规则：提前在store中初始化好所需的属性；使用Vue.set(state.对象,”key”,”value”)添加已初始化对象的属性，实现响应式；Vue.delete(state.对象,”key”)响应式删除对象属性

  • action

    • 异步操作

    • 它的方法中有两个默认参数：context(上下文，相当于箭头函数中的this)对象；payload挂载参数

  • module

    • 当应用变得非常复杂时，store对象就有可能变得相当臃肿，为了解决这个问题，Vuex允许我们将store分割成模块，而每个模块拥有自己的state、mutations、actions、getters等
